## ADDED Requirements

### Requirement: llms.txt generator creates static file
The `scripts/generate-llms-txt.mjs` script SHALL generate a static `public/llms.txt` file listing documentation URLs for LLM consumption.

#### Scenario: Script creates llms.txt in public directory
- **WHEN** the script is executed
- **THEN** it SHALL write `public/llms.txt`

#### Scenario: Script overwrites existing llms.txt
- **WHEN** `public/llms.txt` already exists
- **THEN** the script SHALL overwrite it with fresh content

### Requirement: llms.txt uses absolute URLs
The llms.txt file SHALL use absolute URLs with the base domain `https://ui.kaotypr.site`.

#### Scenario: URLs include full domain
- **WHEN** listing a component documentation file
- **THEN** the URL SHALL be formatted as `https://ui.kaotypr.site/docs/components/accordion.md`

### Requirement: llms.txt lists all generated markdown files
The llms.txt file SHALL list all `.md` files generated by the MDX-to-MD conversion script.

#### Scenario: All component docs are listed
- **WHEN** `public/docs/components/` contains `accordion.md` and `button.md`
- **THEN** llms.txt SHALL list both files with their full URLs

### Requirement: llms.txt includes header
The llms.txt file SHALL begin with a header comment identifying the project.

#### Scenario: Header identifies the library
- **WHEN** llms.txt is generated
- **THEN** the first line SHALL be `# @kaotypr/ui Documentation`

### Requirement: llms.txt runs after MDX-to-MD conversion
The llms.txt generator MUST run after the MDX-to-MD script to ensure all `.md` files exist.

#### Scenario: Generator finds existing .md files
- **WHEN** the script runs
- **THEN** it SHALL scan `public/docs/` for existing `.md` files

### Requirement: pnpm script for llms.txt generation
The package.json SHALL include a `generate:llms` script that runs the llms.txt generator.

#### Scenario: Running generate:llms script
- **WHEN** a developer runs `pnpm generate:llms`
- **THEN** the llms.txt generator script SHALL execute

### Requirement: Combined generate script runs all generators
The package.json SHALL include a `generate` script that runs all generators in the correct order.

#### Scenario: Generate script runs in sequence
- **WHEN** a developer runs `pnpm generate`
- **THEN** the scripts SHALL execute in order: `generate:examples`, `generate:md`, `generate:llms`

### Requirement: prebuild hook runs generators
The package.json SHALL include a `prebuild` hook that runs the combined `generate` script before `next build`.

#### Scenario: Build triggers generation
- **WHEN** a developer runs `pnpm build`
- **THEN** the `prebuild` hook SHALL run `generate` before Next.js build starts
